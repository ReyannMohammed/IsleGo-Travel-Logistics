Introduction  
This document outlines the User and System Requirements for the Automated Client 
Booking and Itinerary Management System at IsleGo Travel & Logistics. The purpose of 
this system is to replace the agency's manual, spreadsheet -based workflows with a 
centralized, cloud -based platform. By automating itinerary creation, coordinating 
vendor payments  and providing a collaborative workspace, the system aims to reduce 
client response times, eliminate scheduling errors, and support the agency's goal of 
scaling booking capacity by 30%.  
 
 User Requirements  
User requirements define what the different stakeholders (Travel Coordinators, 
Management  and Clients) need the system to do in order to  perform their tasks 
effectively.  
 
Travel Coordinators  
Travel Coordinators are the primary internal users of the system. They require tools that 
streamline the planning and execution of complex group vacations.  
•Visual Task Management: Coordinators must be able to view project timelines, task 
dependencies  and milestones through a visual interface (such as a Gantt chart or 
Kanban board).  
•Automated Itinerary Generation: The system must allow coordinators to automatically 
generate professional itinerary drafts based on inputted booking data  and reducing 
manual formatting time.  
•Vendor Coordination: Coordinators must be able to track vendor confirmation 
deadlines, payment schedules (hotels, transport, tours)  and contract statuses within 
specific travel projects.  
•Centralized Client Data: Coordinators require immediate access to a secure, 
centralized repository containing client preferences, passport details  and historical 
booking data.  
 
 
 
 

Management  
Management requires oversight capabilities to ensure operational efficiency and to 
monitor the agency's capacity.  
•Real-Time Dashboard: Management must have access to a high -level dashboard 
providing real -time visibility into the status of all active vacation projects.  
•Resource Allocation: The system must allow management to view staff workloads and 
operational capacity to ensure that no customer requests are declined due to 
mismanaged resources.  
•Performance Tracking: Management needs the ability to track key performance 
indicators, specifically client response times and the frequency of scheduling errors.  
 
Clients  
Clients interact with the outputs of the system and require timely, accurate  and 
professional communication.  
•Professional Documentation: Clients must receive fast, accurate, and professional -
looking itinerary drafts.  
•Real-Time Updates: Clients require automated or rapid updates when booking details, 
schedules, or vendor confirmations change.

standards the software must meet to satisfy the user requirements. These are divided 
into Functional and Non -Functional requirements.

must support.

 Non-Functional Requirements  
Non-functional requirements define system attributes such as performance, usability, 
and security.  
Requirement ID  Attribute  Description

confirmation times from

NFR-02 Accuracy  The system's automated 
validation and centralized

Hardware and Software Specifications  
To ensure the successful implementation , the following technical specifications are 
defined for the booking platform.  
 Software Specifications  
•Operating System: Compatible with Windows 10/11 and macOS for internal staff use.  
•Web Browser: Optimized for the latest versions of Google Chrome, Microsoft Edge, and 
Safari.  
•Platform: Low -code SaaS Project Management and CRM platform (e.g., TeamGantt, 
Monday.com  or similar cloud -based booking solution).  
•Integration: Capability to integrate with standard email clients (Outlook/Gmail) and 
calendar applications for automated notification syncing.  
•Database: Cloud -hosted SQL or NoSQL database provided by the SaaS vendor with 
automatic daily backups.  
 Hardware Specifications  
•Internal Workstations: Standard office laptops or desktops with a minimum of 8GB 
RAM and an i5 processor (or equivalent) to handle browser -based multitasking.  
•Network Infrastructure: Stable high -speed internet connection (minimum 10 Mbps 
upload/download) to ensure real -time synchronization of the cloud platform.  
•Mobile Access: Smartphones or tablets (iOS/Android) for travel coordinators to access 
project statuses while in the field or during eco -tourism excursions.  
 Conclusion  
These requirements have been developed to directly address the operational 
inefficiencies identified at IsleGo Travel & Logistics. By focusing on automated itinerary 
generation and centralized data management, the proposed system provides a robust 
solution that is both technically feasible and economically justified.
